JS常识 ​

队列，栈，堆 ​

  • 队列：执行方式是先进先出，例如事件队列，js在执行的的时候会把异步事件插入到事件队列。可以把他想象成通路。
  • 栈：执行方式是先进后出，例如声明变量是，输出的是最后赋值的结果，一般存储基础类型数据，和引用类型数据的指针。
js
function c(d) {
   console.trace(d);
   debugger
}
function a(d) {
   b(d);
   debugger
}
function b(d) {
   c(d);
   debugger
}
a(3);    先把abc依次放入栈，执行顺序 c-->b--->a    执行后释放的过程
  • 堆：一般存储引用类型的数据。

常见的设计模式 ​

  1. 单例模式 确保一个类只有一个实例，并提供一个全局访问点来访问这个实例。
  2. 工厂模式 提供了一种创建对象的方式，而无需指定具体的类。它定义了一个创建对象的接口，但让子类决定实例化哪一个类

bind和call与apply的区别 ​

三者都是改变函数this的指向。 bind返回的是一个新的函数，可以在某个时机手动执行。可以预设一些参数

js
function greet(greeting, punctuation) {
  console.log(greeting + ', ' + this.name + punctuation);
}
 
const person = {
  name: 'Alice'
};
 
// 创建一个新的函数，该函数绑定了 person 对象作为 this，并预置了 'Hello' 作为第一个参数
const boundGreet = greet.bind(person, 'Hello');
 
// 调用新函数，只需传递剩余的参数（这里是 punctuation）
boundGreet('!');  // 输出: Hello, Alice!

而call和apply,会立即执行，返回的是执行结果。call以参数列表形式传递，call以数组形式传递

js
function greet(greeting, punctuation) {
  console.log(greeting + ', ' + this.name + punctuation);
}
 
const person = {
  name: 'Alice'
};
 
greet.call(person, 'Hello', '!');  // 输出: Hello, Alice!

greet.apply(person, ['Hello', '!']);  // 输出: Hello, Alice!

深拷贝和浅拷贝的理解 ​

  1. 通过JSON.parse(),json.stringify进行转化
  2. 浅拷贝会拷贝引用，深拷贝不会拷贝引用。 Object.prototype.toString.call({}).slice(8,-1) 获取准确数据类型
js
// 深拷贝
function copy(source) {
    var o
    if (typeof source === 'object' && source !== null) {
        var obj =Array.isArray(source) ? [] : {}
        for (var i in source) {
            obj[i] = typeof source[i] === 'object' ? copy(source[i]) : source[i]
        }
        o = obj
    } else {
        o = source
    }
    return o
}

组件化和模块化 ​

组件化是从UI层面进行划分的，模块化是从JS层面进行划分的
组件化：对功能进行拆分，组件进行封装，增加组件的通用性，增强代码的可读性，降低维护成本。
模块化：避免代码污染和命名冲突，方便依赖关系关系，提高了代码的复用率，降低了维护成本。

对javascript执行任务的了解 ​

js任务分为三种,同步任务>微任务>宏任务（将所有区块抽象化在自上而下执行），
同步任务是自上而下执行的。
宏任务：插入到下一个任务的最顶端。
微任务：在当前任务(同步任务)的最后执行。promise,async await 属于微任务 其它异步基本属于宏任务。

弹性盒属性有哪些 ​

1. 父级属性
*flex-direction(row) 属性决定主轴的方向（即项目的排列方向）

css
.box {
  flex-direction: row | row-reverse | column | column-reverse;
}

*flex-wrap(nowrap) 内容一行排不下是否换行

css
.box{
  flex-wrap: nowrap | wrap | wrap-reverse;
}

flex-flow(row nowrap) flex-direction和flex-wrap简写

css
.box {
  flex-flow: <flex-direction> || <flex-wrap>;
}

*justify-content(flex-start) 属性定义了项目在主轴上的对齐方式

css
.box {
  justify-content: flex-start | flex-end | center | space-between | space-around;
}

*align-items(stretch) 属性定义项目在交叉轴上如何对齐

css
.box {
  align-items: flex-start | flex-end | center | baseline | stretch;
}

*align-content(stretch) 属性定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用 注:align-items优先级高于align-content

css
.box {
  align-content: flex-start | flex-end | center | space-between | space-around | stretch;
}

2. 子集属性
order(0) 属性定义项目的排列顺序。数值越小，排列越靠前，默认为0。

css
.item {
  order: <integer>;
}

*flex-grow(0) 属性定义项目的放大比例，默认为0，即如果存在剩余空间，也不放大。 注:设置有值时设置宽度不生效。自适应剩余空间

css
.item {
  flex-grow: <number>; /* default 0 */
}

flex-shrink 属性定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。

css
.item {
  flex-shrink: <number>; /* default 1 */
}

flex-basis 属性定义了在分配多余空间之前，项目占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即项目的本来大小。

css
.item {
  flex-basis: <length> | auto; /* default auto */
}

flex属性是flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto。后两个属性可选。

css
.item {
  flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]
}

*align-self 属性允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch。

css
.item {
  align-self: auto | flex-start | flex-end | center | baseline | stretch;
}
