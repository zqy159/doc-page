Nginx ​

指定目录 ​

bash
mkdir /usr/local/nginx
cd /usr/local/nginx

安装依赖（已安装忽略） ​

CentOS Linux 发行版 (CentOS 7.6)

bash
yum -y install gcc zlib zlib-devel pcre-devel openssl openssl-devel

安装 wget（已安装忽略） ​

bash
sudo yum install wget -y

下载并解压 Nginx ​

访问 Nginx 官网（ https://nginx.org/download/）下载最新稳定版的 Nginx 源码包。以下以 Nginx1.13.7 版本为例

bash
wget http://nginx.org/download/nginx-1.13.7.tar.gz 
tar -xvf nginx-1.13.7.tar.gz

编译和安装 Nginx ​

bash
cd nginx-1.13.7  # 进入解压后的目录  
./configure --prefix=/usr/local/nginx --with-http_stub_status_module --with-http_ssl_module  # 配置Nginx安装路径和所需模块  
make  # 编译Nginx源码  
make install  # 安装Nginx

启动 Nginx ​

bash
/usr/local/nginx/sbin/nginx  # 启动Nginx

验证安装 ​

如果 Nginx 安装成功并正在运行，你应该会看到 Nginx 的默认欢迎页面或相应的 HTTP 响应。

bash
curl http://localhost:80

一键装机 ​

bash
yum -y install gcc zlib zlib-devel pcre-devel openssl openssl-devel

rm -rf "$directory_path"
mkdir "$directory_path"
cd "$directory_path"

# 编译 Nginx
make

make install
# 干掉原先的80进程
sudo fuser -k 80/tcp

/usr/local/nginx/sbin/nginx -c /usr/local/nginx/conf/nginx.conf

nginx 在 windows 系统如何应用 ​

官网下载 nginx 的 zip 压缩包 http://nginx.org/download/nginx-1.13.7.zip 解压后去到首个目录执行 nginx.exe

在浏览器执行 http://localhost 便可访问

  1. 目录手动执行的 nginx,或者重复执行,需要通过任务管理器终止或者直接杀死所有 nginx 进程
bash
taskkill /F /IM nginx.exe

2.直接在 cmd 中通过命令执行 nginx.exe,可以通过普通命令终止,以及重置

bash
nginx -s stop
nginx -s reload

设置 Nginx 开机自启动（可选） ​

bash
sudo vi /etc/systemd/system/nginx.service

在编辑器中输入以下内容：

plain
[Unit]
Description=The NGINX HTTP and reverse proxy server
After=network.target

[Service]
Type=forking
ExecStart=/usr/local/nginx/sbin/nginx
ExecReload=/usr/local/nginx/sbin/nginx -s reload
ExecStop=/usr/local/nginx/sbin/nginx -s stop
PrivateTmp=true

[Install]
WantedBy=multi-user.target

保存并退出编辑器后，运行以下命令以重新加载 systemd 配置，并使 Nginx 服务设置为开机自启动：

bash
sudo systemctl daemon-reload  
sudo systemctl enable nginx

然后重启下试试

nginx 常见命令（拓展） ​

/usr/local/nginx/sbin/nginx 为 nginx 可执行文件的目录

启动(-start)

bash
sudo systemctl start nginx  #系统服务命令
或者
sudo /usr/local/nginx/sbin/nginx   #可执行文件中操作

停(-stop)

bash
sudo systemctl stop nginx  #系统服务命令
或者
sudo /usr/local/nginx/sbin/nginx -s stop   #可执行文件中操作

重载(-reload)

bash
sudo systemctl reload nginx  #系统服务命令
或者
sudo /usr/local/nginx/sbin/nginx -s reload #可执行文件中操作

查看版本(-v)

bash
sudo /usr/local/nginx/sbin/nginx -v

测试运行(-t)

bash
sudo /usr/local/nginx/sbin/nginx -t

查看 nginx 是否正常启动

bash
ps -ef | grep nginx

杀死进程

bash
kill -9 进程号

80 端口被占用

bash
fuser -k 80/tcp

bug 记录 ​

  1. 重启时如果出现报错：nginx: [error] open() ＂/usr/local/nginx/logs/nginx.pid＂ failed。则运行：
bash
 /usr/local/nginx/sbin/nginx -c /usr/local/nginx/conf/nginx.conf

再次启动即可！
