Mysql ​

指定目录 ​

bash
mkdir /usr/local/mysql
cd /usr/local/mysql

安装wget（已安装忽略） ​

bash
sudo yum install wget -y

下载并安装MySQL的yum源 ​

CentOS Linux 发行版 (CentOS 7.6)

访问MySQL官网（https://dev.mysql.com/）下载对应版本的MySQL yum源RPM包。以下以MySQL 8.0为例，但请注意，根据您的需求，您可能需要下载其他版本的RPM包。

bash
wget https://dev.mysql.com/get/mysql80-community-release-el7-5.noarch.rpm  
sudo yum install mysql80-community-release-el7-5.noarch.rpm -y

如果您需要安装MySQL 5.7而不是MySQL 8.0，可以使用以下命令禁用MySQL 8.0的yum源并启用MySQL 5.7的yum源：

bash
sudo yum install yum-utils -y  
sudo yum-config-manager --disable mysql80-community  
sudo yum-config-manager --enable mysql57-community

安装mysql服务器 ​

使用yum安装MySQL服务器。

bash
sudo yum install mysql-community-server -y

启动MySQL服务 ​

安装完成后，启动MySQL服务。

bash
sudo systemctl start mysqld

关闭MySQL服务 ​

bash
sudo systemctl stop mysql

重启MySQL服务 ​

bash
sudo systemctl restart mysql

查看MySQL服务状态 ​

使用以下命令查看MySQL服务的状态，确保它正在运行

bash
sudo systemctl status mysqld
或者
sudo service mysqld status

设置开机自启 ​

bash
sudo systemctl enable mysql   # 启用开机自启
sudo systemctl disable mysql  # 禁用开机自启

获取MySQL的临时密码 ​

MySQL安装完成后，会生成一个临时的root密码，该密码存储在MySQL的日志文件中。

bash
sudo grep 'temporary password' /var/log/mysqld.log

登录MySQL并修改root密码 ​

使用上一步的临时密码登录MySQL。

bash
mysql -u root -p

然后，使用以下SQL命令修改root密码。请注意，MySQL 5.7及更高版本对密码策略有严格要求，您可能需要设置一个符合策略的密码, 例如（Qwer@1234）。

sql
ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';  
FLUSH PRIVILEGES;

配置mysql(可选) ​

编辑MySQL的配置文件（通常位于/etc/my.cnf或/etc/mysql/my.cnf），以调整MySQL的行为。

允许远程连接（可选） ​

如果您需要从其他计算机连接到MySQL服务器，您可能需要修改MySQL的用户权限，允许远程连接。（出于安全考虑，通常不建议使用root用户进行远程连接。）

sql
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY '密码' WITH GRANT OPTION;  
FLUSH PRIVILEGES;

数据库库操作 ​

连接数据库（当前服务器下数据库地址可以忽略）

sql
mysql -h数据库地址 -u用户名 -p用户密码
mysql -u root -p  #服务器内访问

查看数据库（显示当前MySQL服务器上的所有数据库）

sql
show databases;

创建数据库（创建一个新的数据库）

sql
CREATE DATABASE 数据库名;

删除数据库（删除指定的数据库）

sql
drop database <数据库名>;

使用数据库（选择并切换到指定的数据库）

sql
use <数据库名>;

查看当前使用的数据库（选择并切换到指定的数据库）

sql
select database();

数据库备份与恢复（当前服务器下数据库地址可以忽略）

备份

sql
mysqldump -h 数据库地址-u root -p 数据库名>dbname_backup.sql

恢复

sql
mysql -h 数据库地址 -u root -p 数据库名< dbname_backup.sql

数据表操作 ​

查看当前数据库中的表（显示当前数据库中所有的表）

sql
show tables;

创建表（创建一个新的数据表，并定义表的字段和属性）

sql
CREATE TABLE 表名 (字段名 数据类型 [字段属性], ...);

删除表（删除指定的数据表）

sql
DROP TABLE 表名;

查看表结构（显示表的字段信息和结构）

sql
DESC 表名; 或 SHOW COLUMNS FROM 表名;

修改表结构

sql
添加字段：ALTER TABLE 表名 ADD 字段名 数据类型 [字段属性];
删除字段：ALTER TABLE 表名 DROP COLUMN 字段名;
修改字段：ALTER TABLE 表名 MODIFY COLUMN 字段名 新数据类型 [新字段属性]; 或 ALTER TABLE 表名 CHANGE 旧字段名 新字段名 新数据类型 [新字段属性];
修改表名：ALTER TABLE 原表名 RENAME TO 新表名;

数据操作 ​

插入数据（从表中查询符合条件的数据，并支持排序、分组和限制返回记录数等功能）

sql
INSERT INTO 表名 [(字段名1, 字段名2, ...)] VALUES (值1, 值2, ...);

查询数据（从表中查询符合条件的数据，并支持排序、分组和限制返回记录数等功能）

sql
SELECT 字段名1, 字段名2, ... FROM 表名 [WHERE 条件] [ORDER BY 字段名 [ASC|DESC]] [GROUP BY 字段名] [HAVING 条件] [LIMIT 数量];

更新数据（更新表中符合条件的记录）

sql
UPDATE 表名 SET 字段名1=新值1, 字段名2=新值2, ... WHERE 条件;

删除数据（删除表中符合条件的记录）

sql
DELETE FROM 表名 WHERE 条件;

数据库用户操作 ​

注解：localhost换成%,支持远程访问，否则只支持当前IP，如没必要不要去支持。new_user（新用户名），my_database（数据库名）。 创建新用户

sql
CREATE USER 'new_user'@'localhost' IDENTIFIED BY 'my_strong_password';

分配所有权限

sql
GRANT ALL PRIVILEGES ON my_database.* TO 'new_user'@'localhost';

分配某些权限（SELECT,INSERT,UPDATE,DELETE,CREATE,DROP ）

sql
GRANT SELECT, INSERT ON my_database.* TO 'new_user'@'localhost';

允许远程连接（将localhost变成%）

sql
GRANT ALL PRIVILEGES ON *.* TO 'new_user'@'%' IDENTIFIED BY '密码' WITH GRANT OPTION;

刷新权限（权限变更大部分情况都需要刷新权限，当你使用GRANT或REVOKE语句时，MySQL会自动刷新权限）

sql
FLUSH PRIVILEGES;

注意事项 ​

1.安全性：在创建用户时，请确保使用强密码，并遵循最佳的安全实践。

2.主机名：在CREATE USER语句中指定的主机名决定了用户可以从哪些主机连接到MySQL服务器。使用'%'作为主机名表示用户可以从任何主机连接。然而，出于安全考虑，通常建议限制用户只能从特定的主机或IP地址连接。

3.权限管理：为新用户分配权限时，请遵循最小权限原则，即只授予用户执行其任务所需的最低权限。这有助于减少潜在的安全风险。
