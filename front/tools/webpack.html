wepack ​

什么是 webpack? ​

webpack 是一个现代 JavaScript 应用程序的静态模块打包器(module bundler)。当 webpack 处理应用程序时，它会递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成一个或多个 bundle。

为什么选择 webpack? ​

1.grunt 和 gulp 是基于任务和流的。找到一个（或一类）文件，对其做一系列链式操作，更新流上的数据， 整条链式操作构成了一个任务，多个任务就构成了整个 web 的构建流程

2.webpack 是基于入口的。webpack 会自动地递归解析入口所需要加载的所有资源文件，然后用不同的 Loader 来处理不同的文件，用 Plugin 来扩展 webpack 功能 webpack 与前者最大的不同就是支持代码分割，模块化（AMD,CommonJ,ES2015），全局分析

webpack 构建流程 ​

1.初始化参数：从配置文件和 shell 语句中读取并合并执行参数。
2.开始编译：根据第一步的参数初始化 conmpiler 对象，加载所有配置的插件，执行对象的 run 方法开始进行编译
3.确定入口：根据配置中的 entry 找出所有的入口文件
4.编译模块：从入口文件出发，递归找出所有的依赖模块,调用配置的 loader 对模块进行处理。
5.输出资源：根据入口文件模块之间的关系组成一个个包含多个模块的 chunk(根据输出的配置，进行分割)，最后在把内容写入到指向的文件系统中

工程化使用 ​

说明：打包时通过 mode 属性来区分生产环境和开发环境进行不同的配置

plain
├── build
│   ├── webpack.base.js
│   ├── webpack.dev.js
│   └── webpack.prod.js
├── src
│   └── index.js
├── public
│   ├── index.html

/build/webpack.base.js

js
const path = require('path');
// 开发环境
const dev = require('./webpack.dev');
// 生产环境
const prod = require('./webpack.prod');
//需要安装html-webpack-plugin生成html模板
const HtmlWebpackPlugin = require('html-webpack-plugin');
const base = {
    // 基础配置
    entry:'./src/index.js',
    output:{
        filename:'bundle.js',
        path:path.resolve(__dirname,'../dist')
    }，
    plugins:[
        new HtmlWebpackPlugin({
        template: path.resolve(__dirname, '../public/index.html'),//模板路径
        filename: 'index.html',//自动生成的HTML文件的名称
        hash: true, // 在引用资源的后面增加hash戳
        minify: {
            removeAttributeQuotes: true // 删除属性双引号
        }
     }),
    ],
    resolve: {
        extensions: ['.js', '.vue'],
    },
}
module.exports = (env) =>{
  if (env === 'development') {
    return { ...base, ...dev }
  } else {
    return { ...base, ...prod }
  }
}

loader 的使用 ​

说明：作用是把原文件内容通过 loader 中的规则转换成新内容，类似于文件转换器。 一般定义在 module 中的 rules 中，loader 的执行顺序是从下至上，从右向左 实际就是在数组中从后往前执行。

自定义 loader ​

plain
├── loader
│   ├── uploadLoader.js

/loader/uploadLoader.js

js
module.exports = function loader(source) {
  console.log(source, this.query);
  //source是文件内容，query是options中的参数
  return source.replace(/&/g, "符号"); //返回处理后的内容
};

/build/webpack.base.js (module 的 rules 引用 loader)

js
module: {
    rules: [
        {
        test: /\.js$/, //匹配JS 文件
        include: path.resolve(__dirname, 'src'),  // 仅处理 src 目录
        exclude: /node_modules/,    // 跳过 node_modules
        use: [
            {
            loader: path.resolve(__dirname, '../loader/uploadLoader.js')
            }
        ]
        },
    ],
}

常见 loader ​

1.处理 css 文件（npm install style-loader css-loader --save-dev） 说明：import './index.css'; 在打包时是无法解析的需要经过 loader 的处理 webpack.base.js

js
// 处理css文件(需要安装style-loader@1.3.0,css-loader@5.2.7 配合webpack4)
module: {
    rules: [
      {
        test: /\.css$/,
        use: ["style-loader", "css-loader",],
        include: [path.resolve(__dirname, '../src')],
      },
    ],
}

2.处理 sass 文件（npm install sass@latest sass-loader@10 --save-dev） Node.js 16.x 需要 node-sass@6.x。 Node.js 18.x 需要 node-sass@7.x 或更高版本。

js
// 处理sass文件(需要安装style-loader,css-loader,sass-loader，sass)
module: {
    rules: [
        {
            test: /\.scss$/,
            use: ["style-loader", "css-loader", 'sass-loader'],
            include: [path.resolve(__dirname, '../src')],
        }
    ],
}

3.处理图片（npm install file-loader@3.0.1 --save-dev） 说明：import avatar from './img/avatar.png';在打包时是无法解析的需要经过 loader 的处理

js
module: {
    rules: [
      // 处理二进制文件(需要安装file-loader)
      {
        test: /woff|ttf|eot|svg|otf/,
        use: {
          loader: 'file-loader'
        }
      },
      // 处理图片
      {
        test: /\.(gif|png|jpe?g|svg)$/i,
        use: [
          // 将满足条件的图片转成base64如果未满足会自动调用file-loader(需要安装url-loader@1.1.2)
          {
            loader: 'url-loader',
            options: {
              limit: 100 * 1024,  //100M以下进行转化
              name: `img/[name].[ext]`
            }
          },
          // 处理图片，将图片打包返回打包后路径(需要安装file-loader@3.01)
          {
            loader: 'file-loader',
            options: {
              name: `img/[name].[ext]`,
            }
          },
          // 压缩图片(需要安装image-webpack-loader@6.0.0)
          {
            loader: 'image-webpack-loader',
            options: {
              mozjpeg: {
                progressive: true,
                quality: 65
              },
              pngquant: {
                quality: [0.65, 0.9],
                speed: 4
              },
              gifsicle: {
                interlaced: false
              },
              webp: {
                quality: 75
              }
            }
          },
        ],
      }
    ],
}

现代浏览器或 Node.js 14+ 原生支持 ES6+ 语法，无需 Babel 转换。

plugin 的使用 ​

说明：扩展 webpack 的功能，让 webpack 具有更多的灵活性

自定义 plugin ​

plain
├── plugin
│   ├── uploadPlugin.js

plugin/uploadPlugin.js

js
module.exports = uploader;
function uploader(config) {
  console.log("获取plugin内容:", config); //获取plugin内容: { s: 1 }
}
uploader.prototype.apply = function (compiler) {
  // webpack编译完成后执行
  compiler.hooks.done.tap("uploader", function () {
    console.log("执行！！");
  });
};

webpack.base.js (module 的 rules 引用 loader)

js
 plugins: [
    // 自定义插件
    new uploadPlugin({
      s: 1
    }),
  ],

  // 拷贝文件

3.gzip 压缩（npm install compression-webpack-plugin --save-dev） 说明：gzip 压缩(需要引入 compression-webpack-plugin 插件)一般是通过服务器进行压缩，视具体情况而定。

js
plugins: [
    new CompressionPlugin({
    algorithm: "gzip",
    test: new RegExp("\\.(" + ["js", "css"].join("|") + ")$"),
    threshold: 10240,
    minRatio: 0.8,
    });
],
